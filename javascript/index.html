1. Integrating JavaScript: Efficiency Principles

The method used to add JavaScript directly impacts user experience through page load performance.

  • Code Separation: Keeps HTML and logic distinct.
  • Maintainability: Simplifies updates and debugging.
  • Performance Caching: External files can be stored in the browser cache to speed up subsequent visits.

Modern standards favor placing <script> tags in the <head> with the defer attribute to ensure non-blocking script loading.

2. Variables and Data Types

Variables are named containers for managing data. The choice between let and const signals intended behavior:

Keyword Core Principle
let For values expected to be updated or reassigned.
const For fixed values that should not change.

Core Data Types:

  • String: Sequences of characters in quotes.
  • Number: Numerical values (integers or decimals).
  • Boolean: Logical true or false.
  • Null: Intentional assignment of "no value".

3. Working with Numbers

JavaScript uses standard operators for calculation: +, -, *, and /.

Operator Precedence: JavaScript follows standard mathematical rules (Division and Multiplication first, then Addition and Subtraction). Use parentheses () to force a specific sequence.

4. Manipulating Text

Strings use zero-based indexing, where the first character starts at index 0.

Essential Methods:

  • .length: Reveals character count.
  • .trim(): Removes whitespace from ends.
  • .slice(): Extracts a substring.
  • .replace(): Swaps character sequences.

Note: String methods are immutable; they return new strings rather than changing the original.

5. Dynamic Output

Template Literals: Enclosed in backticks (`), these allow variables to be embedded directly using ${variableName}. This is the modern, preferred alternative to concatenation for readability and maintenance.
